FUNDAMENTOS & LÓGICA DE PROGRAMAÇÃO

CONCEITO 01
FLUXO DE DADOS: E → P → S

Todo sistema computacional segue o ciclo Entrada → Processamento → Saída. Pense assim: você digita um número (entrada), o programa calcula (processamento) e exibe o resultado (saída).

📥 ENTRADA: Console.ReadLine()
⚙️ PROCESSAMENTO: nota * 2
📤 SAÍDA: Console.WriteLine()

💡 LOGICLAN

Antes de escrever código, desenhe o fluxo em papel. Identificar E-P-S elimina 70% dos erros de lógica.

CONCEITO 02
PSEUDOCÓDIGO & FLUXOGRAMA

Pseudocódigo é o "rascunho" do algoritmo em linguagem próxima ao humano. Fluxograma é a versão visual. Ambos vêm ANTES do código real.

PSEUDOCÓDIGO
INÍCIO
  // Entrada
  LEIA nota
  // Processamento
  resultado ← nota * 2
  // Saída
  ESCREVA resultado
FIM

💀 ALGOREVILS

Não me tragam código sem pseudocódigo! É como construir uma casa sem planta. IMPROVISAÇÃO É AMADORISMO.

CONCEITO 03
VARIÁVEIS & TIPOS DE DADOS

Variáveis são espaços na memória RAM com nome, tipo e valor. Em C#, o tipo é declarado explicitamente (tipagem estática). Cada tipo define quantos bytes são reservados.

C#
// int: inteiros -2,147,483,648 até 2,147,483,647 (32 bits)
int idade = 25;
int pontos = -100;

// long: inteiros maiores (64 bits)
long populacaoBrasil = 215000000L;

Console.WriteLine($"Idade: {idade}, Pontos: {pontos}");
C#
// double: decimais com precisão dupla (64 bits) — PADRÃO
double altura = 1.75;
double pi = 3.14159265;

// float: menor precisão, sufixo F (32 bits)
float temperatura = 36.5f;

// decimal: financeiro (128 bits, sem erro de ponto flutuante)
decimal preco = 99.99m;
C#
// string: sequência de chars, aspas duplas
string nome = "Fabrício";
string curso = "Sistemas de Informação";

// char: caractere único, aspas simples
char inicial = 'F';

// Interpolação de string (C# 6+)
Console.WriteLine($"Olá, {nome}! Curso: {curso}");
C#
// bool: true ou false
bool aprovado = true;
bool matriculado = false;

// var: tipo inferido pelo compilador (ainda tipagem estática!)
var semestre = 3; // compilador infere int
var universidade = "Unisapiens"; // string

ESTRUTURAS DE CONTROLE

CONDICIONAL 01
IF / ELSE IF / ELSE

Permite que o programa tome decisões. O bloco é executado apenas se a condição for verdadeira.

C#
int nota = 72;

if (nota >= 90)
{
  Console.WriteLine("Excelente! 🏆");
}
else if (nota >= 60)
{
  Console.WriteLine("Aprovado! ✅");
}
else
{
  Console.WriteLine("Reprovado! 💀");
}

💡 LOGICLAN

Sempre coloque as condições da mais restritiva para a mais geral. Do maior para o menor. Senão o >=60 captura antes do >=90!

CONDICIONAL 02
SWITCH / CASE

Ideal quando você testa o mesmo valor contra múltiplas opções. Mais legível que uma cadeia longa de if/else.

C#
int dia = 3;

switch (dia)
{
  case 1:
    Console.WriteLine("Segunda"); break;
  case 2:
    Console.WriteLine("Terça"); break;
  case 3:
    Console.WriteLine("Quarta"); break;
  default:
    Console.WriteLine("Outro dia"); break;
}

💀 ALGOREVILS

Esqueceram o BREAK? WELCOME TO FALL-THROUGH HELL. O C# não deixa, mas não confie nisso. Coloquem o break. Sempre. SEM EXCEÇÃO.

LOOP 01
FOR — REPETIÇÃO CONTADA

Use quando você sabe exatamente quantas vezes vai repetir. Possui inicialização, condição e incremento no cabeçalho.

C#
// Tabela de multiplicação do 5
for (int i = 1; i <= 10; i++)
{
  Console.WriteLine($"5 x {i} = {5 * i}");
}

// Acumulador e Contador
int soma = 0; // acumulador
int count = 0; // contador
for (int n = 1; n <= 100; n++)
{
  soma += n; // acumula
  count++; // conta
}
Console.WriteLine($"Soma: {soma}, Total: {count}");
LOOP 02
WHILE & DO-WHILE

While: repete enquanto condição for verdadeira, pode não executar nenhuma vez. Do-While: executa PELO MENOS uma vez, depois verifica.

C#
// WHILE: valida entrada do usuário
int numero = -1;
while (numero < 0)
{
  Console.Write("Digite positivo: ");
  numero = int.Parse(Console.ReadLine());
}

// DO-WHILE: menu que sempre exibe ao menos uma vez
int opcao;
do
{
  Console.WriteLine("1-Jogar 2-Sair");
  opcao = int.Parse(Console.ReadLine());
} while (opcao != 2);

🧠 LOGICLAN — REGRA MNEMÔNICA

FOR = "Para N vezes". WHILE = "Enquanto condição". DO-WHILE = "Faça pelo menos uma vez, depois enquanto". Grave essa ordem!

TÉCNICA ESSENCIAL
TESTE DE MESA (DRY RUN) — RASTREANDO VARIÁVEIS

O teste de mesa simula manualmente a execução do código linha por linha, registrando o estado de cada variável. É a ferramenta mais poderosa para encontrar bugs de lógica ANTES de compilar.

C# — CÓDIGO PARA SIMULAR
int i = 1, soma = 0;
while (i <= 4)
{
  soma = soma + i;
  i = i + 1;

Console.WriteLine(soma);

MODULARIZAÇÃO — FUNÇÕES & PROCEDIMENTOS

MODULAR 01
VOID — PROCEDIMENTOS (SEM RETORNO)

Procedimentos executam ações sem retornar valor. Use void. Eles são "comandos" — fazem algo mas não produzem resultado para usar em expressões.

C#
// Procedimento: faz algo, não retorna nada
static void ExibirSaudacao(string nome)
{
  Console.WriteLine($"=== Olá, {nome}! ===");
  Console.WriteLine("Bem-vindo ao C# Quest!");
}

// Chamada do procedimento
ExibirSaudacao("Fabrício");
ExibirSaudacao("AlgorEvils");
// Reutilização! Mesmo código, diferentes entradas
MODULAR 02
FUNÇÕES (COM RETORNO)

Funções calculam e retornam um valor. O tipo de retorno substitui o void. O resultado pode ser armazenado ou usado em expressões.

C#
// Função: calcula e RETORNA
static double CalcularMedia(double n1, double n2, double n3)
{
  double soma = n1 + n2 + n3;
  return soma / 3.0;
}

static string ClassificarNota(double media)
{
  if (media >= 90) return "A";
  if (media >= 70) return "B";
  if (media >= 60) return "C";
  return "F";
}

// Uso composto:
double media = CalcularMedia(80, 75, 90);
Console.WriteLine($"Média: {media} → {ClassificarNota(media)}");

🏗️ LOGICLAN — DECOMPOSIÇÃO

Um problema grande → várias funções pequenas. CalcularMedia, ClassificarNota, ExibirResultado... cada uma faz UMA coisa bem. Isso é modularização!

MODULAR 03
BIBLIOTECA SIMPLES DE FUNÇÕES

Agrupe funções relacionadas em uma classe estática — sua própria biblioteca reutilizável.

C# — BIBLIOTECA MATEMÁTICA CUSTOM
public static class MatematicaLib
{
  public static double Area(double base_, double altura)
    => base_ * altura / 2; // expressão lambda

  public static double Hipotenusa(double a, double b)
    => Math.Sqrt(a * a + b * b);

  public static bool EhPrimo(int n)
  {
    if (n < 2) return false;
    for (int i = 2; i <= Math.Sqrt(n); i++)
      if (n % i == 0) return false;
    return true;
  }
}

// Usando a biblioteca:
Console.WriteLine(MatematicaLib.Area(6, 4)); // 12
Console.WriteLine(MatematicaLib.EhPrimo(17)); // True

ESCOPO & PASSAGEM DE PARÂMETROS

ESCOPO 01
VARIÁVEIS LOCAIS vs GLOBAIS (CAMPOS)

O escopo define onde uma variável existe e pode ser acessada. Variável local vive dentro do bloco {}. Campo (global) vive na classe toda.

C#
class Jogo
{
  // CAMPO (visível em toda a classe)
  static int pontuacao = 0;

  static void Pontuar(int valor)
  {
    // LOCAL: só existe aqui dentro
    int bonus = valor * 2;
    pontuacao += bonus; // acessa o campo
  } // "bonus" é destruído aqui

  static void Exibir()
  {
    // pontuacao OK — é campo da classe
    // bonus → ERRO! não existe aqui
    Console.WriteLine(pontuacao);
  }
}

💀 ALGOREVILS

Não poluam o escopo global! Quanto mais local, melhor. Variável global só quando ABSOLUTAMENTE necessário. Efeitos colaterais destroem programas.

ESCOPO 02
POR VALOR vs POR REFERÊNCIA

Esta é a distinção mais crítica em programação. Por valor: cópia — o original NÃO muda. Por referência (ref/out): endereço — o original É modificado.

C# — VALOR vs REFERÊNCIA
// POR VALOR — cópia é passada
static void DobrarValor(int n)
{
  n *= 2; // só a cópia muda!
}

int x = 10;
DobrarValor(x);
Console.WriteLine(x); // ainda 10 !

// POR REFERÊNCIA — endereço é passado
static void DobrarRef(ref int n)
{
  n *= 2; // o ORIGINAL muda!
}

int y = 10;
DobrarRef(ref y);
Console.WriteLine(y); // agora é 20 !
